# Dev & Feedback Project

Project URL: https://github.com/users/ParaphrazeTutorials/projects/1

## Statuses
- Backlog: Unscoped or unprioritized.
- Ready: Scoped and ready to start.
- In Progress: Actively being worked.
- In Review: Awaiting code review/QA.
- Done: Completed and merged/released.
- Blocked: Blocked by dependency/risk.

## Required fields
- Type: Feature | Bug | Feedback.
- Priority: P0 (critical) | P1 (high) | P2 (medium) | P3 (low).
- Area: UI | Data | Infra | Docs.
- Size: S | M | L.
- Status: Backlog | Ready | In Progress | In Review | Done | Blocked.

## Labels
- Type: feature; bug; feedback.
- Area: ui; data; infra; docs.
- Priority: priority:P0; priority:P1; priority:P2; priority:P3.
- Size: size:S; size:M; size:L.
- State: status:blocked.

## Intake
- Bug/feedback: .github/ISSUE_TEMPLATE/bug_feedback.yml (issue form).
- Feature: .github/ISSUE_TEMPLATE/feature_request.yml (issue form).

## Working agreement
- New issues land in Backlog. Triage sets Type, Priority, Area, Size.
- Move to Ready when scoped; In Progress when picked up; In Review on PR; Done on merge/close.
- If blocked, set Status=Blocked and add label status:blocked.
